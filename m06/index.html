Què es NodeJS?

NodeJs és un entorn d’execucions de JavaScritp orientat a esdeveniments asíncrons dissenyat per crear aplicacions network escalables.

Perquè es tan popular?

NODE.JS és un entorn d'execució de JavaScript construït al costat del motor JavaScript V8 de Chrome i utilitza un model I/S impulsat per esdeveniments i sense bloqueig. Chrome V8 és un motor de codi obert creat per al navegador Chrome que compila JavaScript i això ho fa més veloç.

Què puc fer amb NodeJS? Qui l’utilitza?

El funcionament intern de l’execució per a JavaScript Node.js, és força Interessant; En comparació amb les tècniques tradicionals de servei web, on cada connexió (que crea una sol·licitud), genera un nou sub-procés, ocupant la RAM del sistema i regularment maximitzant la quantitat de RAM disponible. També inclou un bucle d’esdeveniments com runtime d’execució al lloc d’una biblioteca.

Node és utilitzada per empreses conegudes com Netflix, Ebay o fins i tot la NASA ja que no és un codi molt complex.

Què és NPM i NPX?

NPM és el gestor de paquets que proporciona per defecte NodeJS i proporciona una manera per a que els desenvolupadors instal·lin paquets tant a nivell global com local. A més és una eina CLI que ajuda a instal·lar paquets i a gestionar versions i les seves dependències. NPX és una eina CLI i el seu propòsit principal és facilitar la instal·lació i la gestió de les dependències allotjades en el registre NPM.

Per a què serveix el paquet "express” de NodeJS?

És un gestor de paquets NPM que permet actualitzar-lo i instal·lar biblioteques, eines i mòduls per utilitzar als programes desenvolupats amb Node.js.

Què és “blocking” i “non-blocking” en NodeJS?

El bloqueig és quan l'execució de JavaScript addicional en el procés de NODE.JS ha d'esperar fins que es completi una operació que no és de JavaScript. El no-bloqueig permet que el programa utilitzat segueixi fent la seva funció de corregir sense la necessitat d’esperar a que un processador acabi la seva execució.

Que es una API? I REST? I “Restful”?

 Una API (Application Programming Interfaces) es un conjunt de protocols i definicions que permet desenvolupar i integrar al software de les aplicacions, permeten la connexió entre dues aplicacions de software per mitjà d’un conjunt de regles.

REST (REpresentational State Transfer) és un estil d'Arquitectura software a l'hora de realitzar una comunicació entre client i servidor que, per exemple, ens permet separar-los. El servei web que afegeix una sèrie de restriccions és denominat RESTful.
